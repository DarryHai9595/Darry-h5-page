不是简单地把一个大 AI 拆成多个小 AI，而是让每个 Agent 都拥有独立的身份、性格、专业能力和团队认知。它们彼此认识，知道该找谁帮忙，就像一支真正的创业团队。

🦞 什么是 OpenClaw？

OpenClaw 是一个开源的 AI Agent 网关框架。它允许你在一个网关下创建和管理多个 AI Agent，每个 Agent 拥有独立的工作空间和身份配置。

  • 统一网关：所有 Agent 共享一个入口，用户不需要分别对接
  • 独立身份：每个 Agent 有自己的 SOUL.md（灵魂）、IDENTITY.md（身份）、技能和行为准则
  • 团队协作：Agent 之间可以互相感知、互相调用，形成协作关系
  • 灵活路由：根据消息内容自动匹配到最合适的 Agent

❹ 秘诀

如何让 AI 拥有"灵魂"？

每个 Agent 的"灵魂"由 4 个文件定义。不是简单的 prompt，而是一套完整的身份系统。

🧠 Agent 身份四层模型

🪦

SOUL.md — 灵魂

最深层的人格定义。包含核心价值观、信念体系、对外部世界的认知、对同事的了解。决定了"这个人是谁"。

🪪

IDENTITY.md — 身份

对外展示的身份信息：名字、角色、性格标签、团队关系。决定了"这个人看起来像谁"。

💓

HEARTBEAT.md — 心跳

日常工作节奏和自动化流程。定时巡检任务、自动触发行为。决定了"这个人每天在做什么"。

🔧

AGENTS.md — 团队手册

团队的完整架构和协作流程。每个成员的能力、触发方式、分工逻辑。决定了"这支团队怎么运作"。

✨ 关键设计理念

拟人化不是噱头，而是功能设计。为什么给 AI 性格很重要？

  • 性格决定输出风格——淑芬说话犀利，文文内敛含蓄，同样的数据结果，两个人的表达方式完全不同
  • 团队认知促进协作——老黑知道"增长素材要找图图做"，管管知道"数据分析派给淑芬"，不需要用户手动协调
  • 信念体系保证一致性——每个 Agent 都有自己的决策原则，不会因为 prompt 漂移而改变行为模式
  • 情感链接提升体验——用户不再是面对一个冰冷的机器，而是在和一群"有血有肉"的伙伴协作
❺ 协作

一个需求，八只虾如何联动？

来看一个真实的场景——当需要出一份月度数据报告时，虾队是怎么协作的。

📊 场景：月度数据报告

用户在知音群对管管说："帮我出一份上个月的月度数据报告。"

Step 1↓

💋 管管接单 → 任务拆解

收到需求后，管管自动拆解成子任务：数据汇总 → 数据分析 → 文案撰写 → 图表设计 → 整合成报告

Step 2↓

🔥 淑芬 → 数据清洗 + 分析

淑芬拿到上月的原始数据，清洗异常值，计算核心指标（DAU、留存、收入等），产出数据分析结果

Step 3↓

✨ 文文 + 🎨 图图 → 并行工作

文文根据淑芬的分析结果撰写报告文案；图图同步制作配套的图表和可视化素材。两条线并行推进，节省时间。

Step 4↓

💋 管管 → 质量审核 + 整合交付

管管审核所有产出物，整合成最终报告，交付给用户。全程用户只需要说一句话。

1

用户只需说一句话

不需要指定"找谁做"、"做什么步骤"。AI 主管自动理解意图、拆解任务、分配到最合适的成员。

2

专业的人做专业的事

每个 Agent 只专注自己的领域。淑芬不会被要求写文案，文文不会被要求跑回归分析——各司其职。

3

Agent 之间自动协作

文文写完文案知道要找图图配图，老黑做增长实验知道要找淑芬看数据。团队关系写进了它们的"灵魂"里。

4

主管兜底，质量有保障

管管负责最终审核和整合，确保输出质量。就像真实团队的项目经理一样。

❼ 实现

如何从零搭建？

技术实现其实不复杂。核心就是配置 + 文件，不需要写什么复杂代码。

1

部署 OpenClaw 网关

安装 OpenClaw，配置 API Key，启动网关。网关是所有 Agent 的统一入口。

2

在配置文件中注册 Agent

在 openclaw.json 中添加 agents 列表，每个 Agent 指定独立的目录和工作空间。这一步把"团队"创建出来。

3

为每个 Agent 写"灵魂文件"

这是最核心也最花时间的一步。用 Markdown 写 SOUL.md、IDENTITY.md、HEARTBEAT.md，定义它的人格、能力、工作节奏。

4

配置团队协作关系

在主管的 AGENTS.md 中写明团队架构、每个成员的能力和触发条件。让 AI 主管知道"谁擅长什么"、"任务该怎么分"。

5

接入通讯渠道，开跑

把网关接入企业微信、飞书、Slack 等通讯工具。现在你可以在群里直接和"虾队"对话了。

💡 关键经验

从实践中总结出的几条关键经验：

  • 身份一致性是第一优先级——所有文件中的名字、性格、关系必须完全一致，否则 Agent 会"认知混乱"
  • 拟人化不是装饰，是功能——性格影响输出风格，团队认知影响协作效率
  • 主管的能力决定了上限——如果主管不擅长拆解任务和分配工作，整个团队就无法有效协作
  • 迭代打磨比一次性设计更重要——先跑起来，再根据实际对话效果不断调整灵魂文件
