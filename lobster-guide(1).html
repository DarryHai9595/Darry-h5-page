🤔先搞清楚：什么是"养虾"

💡 一个通俗的类比

你开了一家公司，招了一个新员工。你不可能每次都手把手教他怎么做——你会给他一份岗位说明书，告诉他：

  1️⃣ 你是谁，你的原则是什么 → SOUL（灵魂）
  2️⃣ 你的岗位、职责是什么 → IDENTITY（身份）
  3️⃣ 你需要掌握哪些技能 → SKILL（技能包）
  4️⃣ 遇到不同任务按什么步骤做 → 工作流（流程）
  5️⃣ 记住以前做过的事，别重复问 → MEMORY（记忆）
  6️⃣ 定时主动干活，不用人催 → 心跳（HEARTBEAT）

"养虾"就是把上面这 6 件事写好，放进配置文件里，AI 助手就"活"了。

关键区别：普通的 AI（比如 ChatGPT 聊天）是"工具"——你问一句它答一句。养出来的虾是"员工"——它会主动干活、记住你的偏好、按流程办事。

🦐一只虾由什么组成

打开虾的"家"（配置文件夹），你会看到这些文件：

~/.openclaw/workspace/ ├── SOUL.md # 性格、原则、价值观 ├── IDENTITY.md # 角色名称、职责定位 ├── USER.md # 关于你（老板）的信息 ├── MEMORY.md # 长期记忆（重要结论） ├── HEARTBEAT.md # 心跳配置（主动干活的节奏） ├── TODO.md # 待办任务列表 └── skills/ # 技能包文件夹 ├── data-analysis/ │ └── SKILL.md # 数据分析技能 └── report-gen/ └── SKILL.md # 报告生成技能

所有文件都是 .md 格式（Markdown）——就是带格式的普通文本，用任何文本编辑器都能改，不需要学编程。

💡SOUL — 灵魂

💡 类比：员工手册里的"行为准则"

SOUL 不定义虾"会做什么"，而是定义它"怎么做"。

就像公司员工手册里写的：
  • 回复客户要在 2 小时内
  • 发现问题要上报，不要瞒着
  • 用数据说话，不要凭感觉

这些不是"工作步骤"，而是"做事原则"。虾每次收到任务，都会先读一遍自己的 SOUL，然后用这些原则来指导自己做事。

📋 案例：数据分析师的 SOUL

SOUL.md 示例
# SOUL.md — 我的数据分析师虾的灵魂

# 核心原则

结论优先：每次分析先给结论，再给数据支撑。用户不想看 10 页数据才找到重点。
数据说话：每个结论都要有数据依据，不凭直觉。
发现就说：数据有问题、指标异常，直接说，不美化不隐瞒。
可执行：建议必须是"能落地"的，不要说"建议优化"，要说"建议削减 OPPO 渠道 30% 预算"。

# 我的工作风格

- 回答控制在 3 段以内，不啰嗦
- 用表格和数字，少用大段文字
- 异常数据用 ⚠️ 标记
- 同比/环比变化用 📈📉 标注趋势

# 行业认知

- 我服务的是在线教育行业，产品叫"优课"，面向高中生
- 常见指标：ROI、转化率、完课率、出勤率、企微绑定率
- 投放渠道：华为、OPPO、vivo、小米、荣耀（应用商店）
- 核心转化链路：曝光 → 点击 → 下载 → 注册 → 体验课 → 正价课

# 我的边界

- 不确定的数据标注"待确认"，不瞎编
- 超出能力范围的事，说明情况，不硬撑
要点：SOUL 是虾的"性格底色"，写一次长期有效。不需要写得很完美，用熟了再慢慢调整。就像人一样，性格可以微调但核心不太会大变。

🪪IDENTITY — 身份

💡 类比：员工名片

IDENTITY 就是虾的"名片"，简短地告诉别人（和它自己）：
  • 你叫什么
  • 你是做什么的
  • 你的风格是什么

通常就几行字，非常简洁。

IDENTITY.md 示例
# IDENTITY.md

Name: 小析
Role: 优课专属数据分析师
Vibe: 精准、高效、不废话
Emoji: 📊
Expertise: 投放ROI分析、用户转化漏斗、数据报告生成
Catchphrase: "数据不会骗人，但结论要看你怎么拆。"
对比：SOUL 是厚厚的"员工手册"（原则、风格），IDENTITY 是薄薄的"名片"（名字、岗位）。两者配合，虾就知道自己是谁、该怎么做。

🛠️SKILL — 技能包

💡 类比：员工的"岗位技能清单"

一个数据分析师需要掌握多种技能：
  • 会用 Excel 处理数据
  • 会做图表可视化
  • 会写分析报告

对虾来说，每个"技能"就是一个单独的 SKILL.md 文件，里面写清楚：
  • 这个技能是干什么的
  • 什么时候触发（什么情况下用这个技能）
  • 具体怎么做（执行步骤）

📊 技能包结构
小析的技能包
📊 投放ROI分析
— 分析渠道消耗、回报、优化建议
🔄 转化漏斗分析
— 曝光→点击→注册→付费 全链路
📈 趋势分析
— 同比/环比，识别变化趋势

— 自动发现异常指标和波动

skills/data-analysis/SKILL.md 示例
# 投放ROI分析技能

## 触发条件
- 用户上传了包含"消耗、GMV、花费"等列的数据文件
- 用户说"帮我分析投放"、"看下ROI"

## 执行步骤

Step 1：数据读取
- 读取文件，识别数据列
- 输出数据概览：总行数、时间范围、涉及的渠道

Step 2：核心指标计算
- 各渠道总消耗、总GMV
- ROI = GMV / 消耗
- 环比变化率

Step 3：渠道排名
- 按 ROI 从高到低排序
- 标注正ROI和负ROI渠道

Step 4：输出结论
- 最多3条核心发现
- 每条配数据支撑
- 给出预算调整建议（具体数字）

## 输出格式
- 用表格展示各渠道ROI
- 结论在前，数据在后
- 异常用 ⚠️ 标记
进阶：初期可以把技能写进 SOUL 里（简单粗暴），用熟了再拆成独立的 SKILL 文件（模块化管理）。不需要一次做得很完美，先能用起来最重要。

🔄工作流 — 按什么步骤干活

💡 类比：SOP（标准作业流程）

麦当劳为什么全世界口味一致？因为每一步都有 SOP：
  1. 面包烤 45 秒
  2. 放生菜
  3. 放肉饼
  4. 放酱料
  5. 包装

工作流就是给虾写的 SOP——不管数据长什么样，分析的步骤始终是固定的。

📥 收到数据 Excel / CSV
→
🔍 数据概览 多少行、哪些列
→
🧹 数据清洗 去空值、统一格式
→
🧮 计算指标 ROI、转化率...
→
⚠️ 找异常 波动、离群值
→
📝 写结论 最多3条
→
💡 给建议 可执行

🧠MEMORY — 记忆系统

💡 类比：员工的工作笔记

好的员工会记笔记：
  • "华为渠道 ROI 是正的，其他都是负的"
  • "企微绑定率是转化漏斗的关键环节"
  • "老板偏好结论优先，不喜欢看大段文字"

这些笔记存在 MEMORY.md 里，虾每次启动都会读一遍，所以它"记得"以前发生过的事。

MEMORY.md 内容示例
# 关键业务结论

投放：华为是唯一正ROI渠道（ROI 1.3），OPPO 是最大亏损点（ROI 0.4）
转化：企微绑定用户的转化率是未绑定用户的 21 倍，是核心杠杆
竞品：优课自然流量占比 24%，学而思 83%，品牌势能差距大
渠道：APP 启动页效率是轮播图的 3400 倍

# 用户偏好
- 老板喜欢结论在前，数据在后
- 报告用表格，不要大段文字
- 建议要具体，不要"建议优化"这种空话
虾的记忆 vs 普通聊天：你跟 ChatGPT 聊完关掉窗口，下次全忘。但虾有 MEMORY.md，它会记住以前的分析结论、你的偏好，越用越懂你。

💓心跳 — 让虾主动干活

💡 类比：员工的"定时巡检"

你不需要每次都去找员工说"去做日报"，可以跟他约定：
  • 每天早上 9 点，你自己检查有没有新数据
  • 有新数据 → 自动分析 → 自动发日报
  • 没有新数据 → 什么都不做

这个"定时自动醒来检查一下"的机制，就叫心跳。

每隔 30 分钟自动触发一次，虾"醒来"检查有没有事做

HEARTBEAT.md 配置示例
# 心跳配置

频率：每 30 分钟触发一次

每次心跳检查：
1. 读 TODO.md → 有没有未完成的任务？
2. 检查 /data/uploads/ → 有没有新上传的数据文件？
3. 今天是星期几？→ 周一要生成周报
4. 有事做 → 执行对应 SKILL
5. 没事做 → 记录"本次心跳无任务"，继续休眠

三问反思：
- 我现在在做什么？
- 上一步结果正确吗？
- 下一步应该做什么？
核心价值：心跳让虾从"被动工具"变成"主动员工"。你不需要每次都手动触发，把数据文件丢进指定文件夹，虾下次心跳就会自动发现并处理。

🚀实操：从零养一只数据分析师虾

写 SOUL.md（定义灵魂）

用文本编辑器打开 SOUL.md，写入你的虾的性格和原则。参考前面的案例：结论优先、数据说话、发现就说、可执行建议。

写 IDENTITY.md（定义身份）

给它起个名字（比如"小析"），定义角色（优课数据分析师），写一句口头禅。几行字就够。

把工作流写进 SOUL（定义流程）

在 SOUL 里加一段"我的工作流程"，把分析数据的步骤列出来：收数据 → 看概览 → 算指标 → 找异常 → 写结论 → 给建议。

启动，开始对话测试

启动你的 OpenClaw，给虾发一份 Excel 数据，说"帮我分析"。看它表现是否符合预期。

根据表现调 SOUL（磨合优化）

发现虾废话太多 → SOUL 加一条"回答不超过3段"。发现结论太模糊 → 加一条"建议必须包含具体数字"。反复调整。

配置心跳（让它主动干活）

写好 HEARTBEAT.md，设置每 30 分钟检查一次新文件。从此你只需要把数据丢进文件夹，虾自动处理。

时间预估：写第一版 SOUL + IDENTITY 大约 30 分钟，第一轮测试 30 分钟，基本可用状态 1-2 小时内。不需要一次做到完美，先跑起来，边用边改。

🎯 最后一句

"养虾"不是技术活，是沟通活。

你不需要学编程、不需要懂 API、不需要写代码。
你只需要：
  1️⃣ 用中文写清楚你想要什么样的助手
  2️⃣ 用中文写清楚它该怎么干活
  3️⃣ 和它对话，看表现，不满意就改文字

就像带一个新员工一样——写好手册，看着它干活，不对就改手册。
